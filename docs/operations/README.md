# 운영 절차

이 디렉터리는 권한이나 외부 서비스가 관련되고 실패 시 재시도 또는 복구가 필요한 운영 절차를 설명합니다.

## 문서

- [릴리스](./release.md): 버전 PR 생성부터 npm 배포까지의 흐름

## 문서 추가 기준

- 배포, 권한 또는 외부 서비스 상태에 영향을 받는 작업
- 실패 시 재시도, 복구 또는 수동 확인이 필요한 작업
- 배포나 권한처럼 외부 상태를 변경하는 CI 또는 GitHub Actions 작업

일상적인 코드·문서·버전 기록 절차는 [개발 가이드](../guides/README.md)에 둡니다. 반복되는 실패는 가장 가까운 운영 문서의 문제 해결 절에 기록합니다.
