# 실행 계획

이 디렉터리는 여러 세션이나 여러 변경에 걸친 작업의 실행 계획을 보관합니다. 작은 작업의 일회성 계획은 대화나 이슈에 남기고 저장소 문서로 만들지 않습니다.

## 작성 기준

다음 중 하나에 해당하면 실행 계획을 작성합니다.

- 여러 패키지 또는 여러 PR에 걸친 변경
- 단계별 마이그레이션과 호환성 관리가 필요한 작업
- 작업 도중 중요한 선택과 검증 결과를 계속 기록해야 하는 작업

## 권장 형식

```md
# 목표

- Status: Active
- Started: YYYY-MM-DD
- Closed: YYYY-MM-DD | N/A
- Scope: 영향을 받는 패키지와 경로

## 성공 기준

완료를 검증할 수 있는 조건

## 단계

- [ ] 작업과 검증 방법

## 결정 기록

작업 중 선택한 방향과 이유

## 진행 기록

날짜별 결과, 실패, 다음 행동

## 종료 결과

- 최종 결과
- 검증 결과
- 남은 작업
```

파일명은 `YYYY-MM-DD-short-title.md` 형식을 사용하고 이 디렉터리에 바로 추가합니다. 상태는 `Active`, `Completed`, `Cancelled` 중 하나로 관리합니다.

완료하거나 취소한 계획은 상태, `Closed`, 종료 결과를 갱신한 뒤 같은 위치에 보존합니다. 문서가 많아지면 완료·취소 계획을 `archive`로 이동하고 해당 디렉터리에 인덱스 `README.md`를 둡니다. 장기적으로 유지할 설계 결정은 계획에만 남기지 않고 [ADR](../decisions/README.md)로 분리합니다.
